GSoC 2025 Zarr: Introduction

Introduction to the Bit-Packing Project of Zarr under NumFOCUS

Side Channel Analysis (SCA) has grown into a critical area of research due to its profound implications for both security and privacy. By examining power consumption from electronic devices, attackers can infer sensitive information. SCA contributes to developing stronger cryptographic protocols, more secure hardware designs, and advanced countermeasures. In addition, better protection against side channel attacks safeguards personal data and financial transactions, underscoring the vital need for ongoing investigation in this field.

A key challenge in large-scale SCA arises from the massive volume of high-resolution measurements needed for thorough security evaluations. These measurements often captured at 10-bit or 12-bit resolution, with large datasets for each time steps. The SCARR framework (Bosland et al. 2024) introduce a high-performance design patterns, combined with a modern file format Zarr and optimized compression, which can substantially reduce data sizes while accelerating key SCA algorithms. This enable SCA procedures such as Signal-to-Noise Ratio (SNR) and Correlation Power Analysis (CPA) to run significantly faster, often outperforming existing frameworks by a factor of two.

Zarr’s design, which allows for on-the-fly compression and efficient random access to large datasets, is ideally suited for SCA research. Integrating specialized bit-packing algorithms into Zarr can further boost performance when handling 10-bit or 12-bit ADC samples, providing both lower storage costs and faster I/O operations. In doing so, the project directly contributes to SCARR’s vision of making side-channel evaluations more scalable and cost-effective, ultimately enabling more comprehensive security testing in SCA research.

This Google Summer of Code project aims to create bit-packing compression method optimized for SCA data in the open source library Zarr. By focusing on an approach that caters to 10-bit or 12-bit ADC samples, the project seeks to find any possible compression method that performs between the baseline of uncompressed data, and LZ4 within HDF5. Through efficient compression, the initiative will help researchers spend more time on analysis and less time on I/O, thereby enabling faster experimentation and a deeper exploration of side channel analysis
